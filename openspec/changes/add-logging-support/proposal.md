# 添加日志支持

## 变更概述
为MCE区块链服务添加全面的日志支持，使用Winston提供结构化日志，包括请求日志、错误跟踪和区块链操作日志。

## 问题描述
当前应用缺乏适当的日志基础设施，导致：
- 难以跟踪请求流程和调试问题
- 无法监控区块链操作
- 缺少系统行为审计
- 无法分析性能指标
- 生产环境故障排查困难

## 解决方案
实现基于Winston的日志系统，包括：
- 生产环境的结构化JSON日志
- 开发环境的控制台日志
- 请求/响应日志中间件
- 错误日志集成
- 可配置的日志级别
- 日志轮转和归档

## 验收标准
1. Winston日志库集成完成
2. 请求/响应日志中间件实现
3. 错误日志与Express错误处理集成
4. 生产环境结构化日志格式
5. 开发友好的控制台日志
6. 可配置的日志级别（error, warn, info, debug）
7. 日志配置管理

## 范围
### 包含内容
- Winston日志器配置
- 请求日志中间件
- 错误日志集成
- 日志级别管理
- 环境特定配置

### 不包含内容
- 日志聚合服务（ELK、Splunk）
- 实时日志监控仪表板
- 高级日志分析
- 外部日志服务集成

## 依赖项
- Winston日志库
- 现有Express应用
- 现有错误处理中间件
- 当前项目结构

## 风险评估
- 低风险：对现有代码库的非侵入性添加
- 中等风险：需要中间件集成
- 风险缓解：全面测试和逐步推出

## 实现说明
- 遵循现有DDD架构模式
- 与当前错误处理集成
- 保持向后兼容性
- 使用TypeScript接口确保类型安全
